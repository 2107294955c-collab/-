# Week 14 - Laikago 四足机器人运动学与物理稳定性调试报告

## 📌 实验目标
本周核心任务是对宇树科技 Laikago 四足机器人在 PyBullet 仿真环境下的落地稳定性与 Trot（小跑）步态进行联合调试。重点解决在纯数学波形驱动下，机器人高频出现的**出生姿态错乱、关节物理打架、落地弹跳翻车**等底层动力学崩溃问题。

---

## 🔍 问题排查与核心痛点

在仿真初期，机器人无论如何修改步态参数，均无法实现平稳落地，运行状态呈现“底盘擦地”或“关节死锁绞死”的异常姿态。通过深度排查，定位到以下两个关键性物理 BUG：

1. **出生旋转姿态错误（Orientation Error）**
   * **现象**：机器人初始化后瞬间在空中发生扭转并摔骨折。
   * **原因**：历史代码中包含 `p.getQuaternionFromEuler([math.pi / 2, 0, math.pi / 2])` 逻辑，在加载 URDF 时强行为机器人注入了 $90^\circ$ 的 Pitch 轴初速度/位姿，导致其侧向砸向地面。
2. **前后腿物理极性反转（Link Polarity Misalignment）**
   * **现象**：施加静态站立角度后，四条腿在腹部下方相互错位穿模。
   * **原因**：Laikago URDF 模型定义中，前腿（LF/RF）与后腿（LH/RH）的大腿（Thigh）与小腿（Calf）关节的物理限位及正方向呈现**镜像反转**。使用统一的正弦波或简化几何角度驱动，会引发前腿与后腿运动相互冲突抵消。

---

## 🛠️ 解决方案与控制架构设计

为了达成**绝对平稳落地且自由移动**的目标，本周控制器进行了工业级的架构重构：

### 1. 工业级官方逆运动学（IK）引擎集成
废弃了简化版的几何正弦计算，直接调用 PyBullet 内置高维动力学解析器 `p.calculateInverseKinematics`。
通过精确绑定机器人的足端 Link（Link ID: 2, 5, 8, 11），在三维空间中为其规划标准的**椭圆步态轨迹**，确保足端在摆动相（Swing Phase）平滑抬起，在支撑相（Stance Phase）完美压紧地面。

### 2. 状态机分段缓冲机制
为了阻断开局下坠时碰撞产生的数值奇点，引入了**两阶段淡入机制**：
* **阶段一：雕塑站立压实（0.0s - 1.0s）**：步态尺度限制为 0。机器人通过高马力（`force=200`）强行保持完美的静态微蹲板凳姿态。使其自然接触地面，吃满重力，完成摩擦力交互。
* **阶段二：步态平滑淡入（1.0s 后）**：运动幅度和控制力矩在 2.5 秒内呈线性缓慢递增（`start_up_scale` 0.0 $\to$ 1.0）。完全隔绝了瞬时大控制量带来的肢体抽搐。
