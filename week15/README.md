# Week 15 - 基于 Tailscale 虚拟局域网的移动端摄像头局域网图传系统搭建与调试

## 📌 实验目标
本周实验重点在于打通**移动端（手机）与本地开发环境（WSL/Windows）之间的跨设备局域网通信**，实现通过手机浏览器调用摄像头并将实时画面流传输至电脑服务器。
- [x] 配置 **Tailscale 虚拟局域网**，实现手机与电脑在非同一物理 Wi-Fi 下的互联互通。
- [x] 运行 `camera_bridge.py` 脚本，搭建基于 HTTPS 协议的本地图传接收服务器。
- [x] 调试移动端浏览器权限与网络策略，解决数据流传输中的阻塞问题。

## 📸 实验核心效果与步骤

### 1. 生成 ArUco 标记
- **配置参数**：字典选择 `4x4 (50, 100, 250, 1000)`，Marker ID 设置为 `0`，尺寸为 `100mm`。
- 通过在线生成器（ArUco markers generator）在电脑屏幕上渲染出特定编码的黑白矩阵标记。

### 2. 移动端跨网络图传与算法检测

- 手机端通过 Tailscale 虚拟局域网（分配 IP 如 `100.77.113.4`）成功穿透网络限制，调用手机摄像头对准屏幕上的标记。

### 3. 最终服务器检测结果
如图所示，系统运行状态完美，后端算法成功捕捉到了通过手机传输过来的图像流，并在画面中精准定位了标记，绿线框选出了边缘，并成功解析输出其 ID 为 **`ID: 0`**。

## 🔍 问题排查与核心痛点（重点踩坑记录）
在部署与联调过程中，系统经历了从“完全无法访问”到“卡在等待状态”的典型网络与权限故障。通过逐级排查，最终定位并解决了以下两个核心问题：

### 1. 移动端无法解析与连接超时（Connection Timeout）
- **💥 现象**：手机端输入网址后，浏览器长时间白屏，最终提示“无法访问该网站”或“连接超时”。
- **🛠 修复方案**：开启手机端 Tailscale App 并确保状态为 `Connected`；在手机浏览器中严格输入带有安全协议及正确端口的完整路径：`https://<Tailscale_IP>:5000`。

### 2. 服务器检测结果始终卡在 `waiting` 状态
- **💥 现象**：手机端虽然成功绕过了隐私警告并打开了网页界面，但电脑服务端的终端日志始终静止在 `waiting`，无法接收到任何图像帧。
- **🛠 修复方案**：在手机浏览器地址栏点击警告图标，将“摄像头”权限手动更改为**“允许”**；强烈建议在移动端使用 **Chrome（安卓）** 或 **Safari（iOS）** 以获得更规范的硬件授信机制。
